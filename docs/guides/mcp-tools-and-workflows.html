The one concept to grasp

What is an "MCP tool"?

MCP lets Claude reach into a real system and do real work. Each tool is one specific action it can take — like "search contacts" or "log a touchpoint."

  • You talk normally. "Who haven't we followed up with?" Claude picks the right tool.
  • It's live data. Tools read and write the same Supabase database the web app uses — changes show up everywhere instantly.
  • It's guarded. Every write runs through the same validation as the app. Bad data is rejected, not saved.
  • It can't go rogue. There are exactly 14 actions. No tool deletes records or does anything destructive.

Adding & merging records

create_or_update_contactAdd a new person, or merge new info into an existing one.
create_or_update_orgSame, for organizations.

The safety rule that matters: merging is fill-empty by default. New facts fill blank fields, touchpoints get appended, relationship types combine — but verified data is never overwritten unless you explicitly say so. Leave the ID off and Claude refuses to create a duplicate of someone who already exists.

📧 From a Gmail thread: a new human you emailed back becomes a fresh contact (status cool, tagged email_sourced); a known contact writing from a new address gets matched by name and merged — never duplicated.

Before the workflows — three things to know

The vocabulary

The relationship ladder

Everyone sits at one of four stages:

cold → cool → warm → active

The goal is to move people rightward over time.

Cadence & "next action"

Each person has a next action + date. When that date passes with nothing done, they go overdue — that's what the health check surfaces. No next action = invisible.

Readable IDs

Records use human IDs, not random codes:

ind_jane_smith
org_brooklyn_arts
evt_spring_gala

You rarely type these — Claude generates them. But seeing one in a reply tells you exactly which record it touched.

Workflow 4 · Periodic

Bulk merge from sign-up sheets

Turn Google Form responses into enriched contacts — additively, never destructively.

  • Match on email first, name as fallback — so existing people get enriched, not duplicated.
  • Fill blanks only. A record that already has an email keeps it. New people come in as warm.
  • Re-run anytime. The Form sheets keep growing; merging again just adds the newcomers.

contacts. 14 new, 9 enriched, 13 unchanged, 0 duplicates.

Two front doors: paste a sheet into the app's Merge spreadsheet mode, or have Claude pull it live via the Drive connection.

Habits that keep it healthy

Five rules of thumb

  • Always leave a next action. A contact with no dated next action is invisible to the dashboard.
  • Log the touchpoint while it's fresh. One sentence to Claude beats a perfect note you never write.
  • Check before you create. "Is this person already in here?" prevents almost every duplicate.
  • Trust fill-empty merging. Re-running a merge or scaffold is safe — it never overwrites verified facts.
  • Describe goals, not tools. Say what you want; let Claude pick from the 14. You never memorize a tool name.
