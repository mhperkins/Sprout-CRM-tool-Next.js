Delivered · 2026-06-19

Fix 1 Preview no longer jumps to top while typing

Each keystroke was swapping the iframe srcDoc, forcing a full reload that reset scroll to 0; the restore flickered and landed short. Now the iframe loads a static shell once and each update swaps only the inner DOM (documentElement.innerHTML) — the window persists, so scroll stays exactly where you're editing. Bonus: images stop re-fetching on every keystroke.

Fix 2 Per-event scroll in repeat sections

Upcoming/Past events shared one section marker, so clicking any event field scrolled to the first card. Each card now carries data-sec-item="upcoming-<i>" / past-<i> and focusScroll(key, idx) jumps to the exact card you're editing.

Fix 3 Same mechanism in the Quick Hit template

The one-off bracket template had no sync wiring. Added inert data-sec markers (headline / recap / next-up / CTA), a keyword resolver mapping each placeholder to its block, and template-aware group helpers so focus→scroll and click→focus both work there too.
