Grant-tool build break

The redeploy surfaced a failure in a different repo (sprout-grant-tool → "sprout-tools"): supabaseKey is required from a module-scope createClient. Lazy-init fix verified green locally; awaiting OK to push to that repo's main.

Why it broke at build time

createClient(url, SUPABASE_SERVICE_ROLE_KEY) ran at module scope, so next build evaluated it during page-data collection — where the service-role key is absent — and createClient throws synchronously on a missing key. Moving the client into the request handler defers it to runtime. The visible export const config warning was a red herring; removed it anyway (Pages-Router leftover the App Router ignores).
