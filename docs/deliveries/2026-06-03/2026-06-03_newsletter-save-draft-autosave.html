Newsletter drafts now save, auto-save, and survive a reload

A Save-draft button, auto-save on navigate-away, and reload persistence for the in-app newsletter editor.
main 5aa8fdc Effort: medium ✓ build passes
💾

Save draft button

Persists to Supabase but stays in the editor. The old "Save" (which closed back to the list) is now Save & close. A new draft adopts its generated id on first save, so repeat saves update the same record instead of duplicating.

🚪

Auto-save on navigate-away

Saves on unmount (← Back, sidebar view switch) and on tab close via beforeunload. Guarded by draftHasContent() so a blank new draft isn't saved, and suppressed on Delete / explicit close so nothing is resurrected or double-written.

🔄

Persist on reload

The open editor session snapshots to localStorage (sprout_nl_editor_v1) on every change and restores once on mount — in a useEffect to dodge a hydration mismatch. A reload mid-edit returns you to the draft intact, even a never-saved one.

🧩

Under the hood

New editor callbacks onSaveStay / onSaveClose / onAutoSave replace the single onSave prop. makeRecord(d) factors the shared id-assign + HTML-bake; refs capture the latest draft + built HTML for the unmount cleanup.

⚠️

The duplicate trap we closed

The new-draft id is a deterministic slug (nl_<month>_roundup). If auto-save wrote a record while the localStorage snapshot still held id:null, returning would regenerate the same base id, find it taken, and bump to _2 — a fresh duplicate every navigate-away/return cycle. Fixed by having onAutoSave write the saved record (with its assigned id) back into the snapshot, so restore returns the id and further saves update in place.
