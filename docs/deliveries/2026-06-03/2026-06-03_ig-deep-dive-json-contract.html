IG Deep-Dive JSON Contract + where CRM research actually runs

Settled the surface split for the Instagram pipeline and shipped a Desktop-readable output contract so deep-dive research lands in the CRM with zero silent data loss.

Docs only No app code change No data change 1 new guide

Max asked Claude Code to scrape recent IG followers and save a triage sheet. The real finding: Claude Code in VS Code can't scrape Instagram — it has no browser bridge. The scrape + deep research run in Claude Desktop + Claude in Chrome; this project is the landing zone (filter, dedupe, import).

🌐Research surface = Desktop + Chrome

Only the Chrome extension can drive the logged-in IG session and read the virtualized followers modal. A plain WebFetch can't log in or render it. So scraping & deep dives live there.

🛬Landing zone = Claude Code (VS Code)

Takes the returned JSON, runs check_existing per handle, merges dupes instead of duplicating, and imports the rest through the hardened Import view.

📄New doc: ig-deep-dive-json-contract.md

Self-contained restatement of the import contract for the research project to consume: the two silent-failure modes, full org + contact examples, the enum table, dual next-action rule, touchpoint shape.

🪤The traps it spells out

Wrong enum = whole record silently dropped. Doc flags arts, musician, photographer, community builder (with a space). Thin accounts → leave out, never fabricate.
