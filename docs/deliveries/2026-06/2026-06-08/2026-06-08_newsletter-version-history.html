Newsletter Version History 📌

Save named snapshots and restore them any time — alongside Save draft.
18981b2 main Effort: medium Files: 2 + changelog build: ✓ passes
📌

Save version button

New button in the editor's action bar. Prompts for an optional name, then commits a snapshot of subject, month, template, all field_values, spotlight, limits + the baked HTML.

🗂️

Version history card

Lists every saved version newest-first with name + timestamp. Each row offers ↩ Restore (behind a confirm, history kept) and 🗑 delete.

🧬

Persisted, no migration

New JSONB-only versions[] on NewsletterSchema, defaults to []. Rides the existing save path and survives reloads.

🎯

Intentional by design

A version is created only by the 📌 button — Save draft, auto-save, and sending do not. Restore loads into the editor; you decide whether to keep it.

🔧

Where it lives

components/CRMManager.jsx — saveVersion / restoreVersion / deleteVersion + the Version history card in NewsletterEditor.  ·  lib/schemas.js — the versions field. Deep-clone on both save and restore so snapshots never share references with the live draft.
