Ask 2 · Newsletter

Where the passphrase lives

It's the NEWSLETTER_SEND_SECRET env var, set in two places:

.env.local → localhost dev only

Vercel (sprout-crm-tool-next-js) → the live app

Each environment validates against its own copy — so edit the live one in Vercel + redeploy.

The "wait, why does it still work?" clarification

The two values drifted apart — and that's fine

.env.local starts russ…, Vercel starts sk_…, yet sending works perfectly. Why? app/api/send/route.js checks the entered passphrase against process.env.NEWSLETTER_SEND_SECRET of whichever environment is running it. The live site only ever compares against the Vercel value.

Bottom line: the sk_… Vercel value is your real working passphrase for the deployed app. The russ… local one is a stale leftover that only matters if you send from npm run dev.
