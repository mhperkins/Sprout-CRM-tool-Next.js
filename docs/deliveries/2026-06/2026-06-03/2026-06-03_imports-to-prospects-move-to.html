With Contacts split into Community / Donors / Prospects, the import needed to know where each person lands. Orgs are trivial (no segment). The answer for people, kept deliberately simple: every import goes to Prospects, then you reclassify by hand in one click.

① Default rule

All imports land in Prospects

prepareImportItem forces segment:"prospect" on every imported individual, overriding whatever the JSON says. A fresh follower scrape is by definition a list of prospects — no decision tree for the research surface to follow.

② Donor cross-check

Auto-upgrade known donors

Before validation, each person is matched against existing CRM donors (the 26 Givebutter donors) by id, email, or @handle. A match flips prospect → donor and shows matched existing donor → Donors in the preview.

③ Move-to control

Reclassify in one click — no modal, no save

A new Bucket: [current] · Move to ▾ line sits at the top of the contact detail panel. Pick another bucket → instant move + toast. Triage now happens right after you read the person, instead of three clicks deep in the Edit modal.

Prospects → Community ／ Donors · one tap, reversible
Heads up

Re-importing resets the bucket

Re-importing an existing contact by id resets them to Prospect unless they match a donor. Intended for the fresh-follower pipe. The IG deep-dive contract docs no longer need to stamp segment — a one-line edit for next time those docs are touched.
