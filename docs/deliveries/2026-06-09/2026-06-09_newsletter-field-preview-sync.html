Field ↔ preview sync 🌱

Focus a section field and the live preview scrolls to that block (and flashes it). Click a block and the editor jumps to its field. The preview is now a pinned panel.
🎯
Forward sync — focus → scroll
Every section field got an onFocus that maps its key to a block group and scrolls the preview iframe to [data-sec=…], with a fuchsia outline flash (~1.4s) so it's obvious which block you're editing.
👆
Reverse sync — click → field
Clicking a block in the preview scrolls the editor to that section's first field and focuses it. Real links and buttons inside a block still work normally.
🏷️
11 blocks tagged
Each block in buildCompact() carries data-sec="<group>" (17 tagged cells). These are inert in email — Gmail/Outlook/Apple Mail ignore unknown attributes, so the sent newsletter is unchanged.
📌
Sticky preview panel
The preview is now position:sticky with internal scroll, so it can move to a block without pushing your field off-screen. Its scroll position survives the per-keystroke reloads.
🐛
Bug caught + fixed in the same session
First pass rendered the preview narrow & compact with a stray horizontal scrollbar. Cause: alignSelf:"flex-start" on the card — in a column flex container that's the horizontal axis, so it collapsed the card to content width and the email reflowed to its phone layout. Removed it → card fills the column again, preview back to full size.
Trade-off noted to Max: the preview changed from "auto-grows to full height, page scrolls" to a pinned panel that scrolls internally to what you're editing — better for editing. A toggle between "follow my edits" and "full-length" view modes is available on request.
