"See more events" button auto-scrolls again

The featured newsletter button always opened a hardcoded Luma link in a new tab — even with the link field empty. Restored its original behavior: scroll down to the Upcoming events section.
File  lib/newsletter.js Effort  diagnose low / fix low Build  ✓ passes 49d6442 · main
🐞 Root cause

The render used tval(v.seeMoreLink, "https://luma.com/..."). tval returns its fallback when a field is blank, so an empty link field still produced the Luma URL plus target="_blank". The May draft's stored value was actually null all along.

🔧 The fix

Blank link → href="#events", scrolls to the Upcoming section (no new tab). Typed URL → links out in a new tab. Editor field relabeled to "(blank = scroll to Upcoming)" so the default is clear.

Button behavior now
link field blank → scrolls to #events (Upcoming) · default
URL typed in → opens that URL in a new tab
⚠️ Email caveat unchanged: Gmail/Outlook strip id anchors, so the in-email scroll may not fire there — the typed-URL fallback covers that case if ever needed.
