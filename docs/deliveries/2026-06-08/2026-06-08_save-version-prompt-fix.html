Fixed: "Save version" crash —
prompt() is not supported
Next.js 16's dev runtime blocks native browser dialogs. The version-history flow used window.prompt and window.confirm; both now run through in-app modals.
🐛 Crash fix Effort · diagnose low / fix low ✓ npm run build passes
🐛 The bug
Clicking 📌 Save version threw a runtime error prompt() is not supported — call stack landed on saveVersion.
🔎 Root cause
Next 16 blocks native prompt / confirm / alert. saveVersion used a prompt to name versions; restoreVersion used a confirm.
📌 Fix · Save version
Now opens a small name modal (defaults to "Version N", autofocused, Enter to confirm). New commitVersion() does the snapshot/persist.
↩ Fix · Restore
Routes through the existing ConfirmModal ("Restore version", non-danger) instead of window.confirm.
🧭 Lesson
Never use native prompt/confirm/alert in this app — use the Modal / ConfirmModal primitives. This was the only place that still slipped through. Both behaviors are otherwise identical to before.
