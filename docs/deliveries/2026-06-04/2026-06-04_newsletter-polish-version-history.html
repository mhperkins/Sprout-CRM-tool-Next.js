Newsletter ✨ Polish is now non-destructive

Clicking Polish used to overwrite your draft in place — one bad rewrite and the original was gone. Now the agent's version appears in an inline card below the field, your original stays put, and every re-polish is kept as a pickable version.
⎇ b4be500 main · pushed Effort: medium ✓ npm run build
🛟 Original preserved

Polish no longer calls apply(). The rewrite lands in a new per-field polishOut state — the field itself is never touched until you choose to use a version.

🗂️ Version history

Each ✨ Re-polish stacks a new numbered version (v1 · v2 · v3 · latest). Liked the first one better? It's still there with its own button.

✓ Use this / ✕ Discard

Use this drops that version into the field and closes the card. Discard clears it with nothing changed. Button relabels to Re-polish once a result exists.

🔌 Both surfaces

Wired into single copy fields (Headline, Intro, Recap…) and per-item repeat fields (each event entry). One polishPanel() renderer, two call sites.
