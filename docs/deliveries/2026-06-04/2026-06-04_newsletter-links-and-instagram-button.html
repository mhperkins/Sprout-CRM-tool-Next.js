"See more events" no longer hijacks the preview — and every link now works

Diagnose · medium Fix · low

Clicking "See more events" in the compact-newsletter preview was reloading the pane and rendering the live CRM Dashboard inside it. Root-caused to the iframe base-URL inheritance, fixed preview-side, then wired up the membership button, footer contact links, and a brand-new Instagram button.

↧ Scroll fix, preview-only

New previewHtml memo injects <base href="about:srcdoc"> so #events scrolls inside the iframe instead of loading the app. Sent HTML untouched. Iframe sandboxed (scripts off, popups on).

🔗 Reliable scroll target

Moved id="events" off a stray <a> (foster-parented out of the table) onto a real <tr id="events">.

🎟 Buttons + footer links

Membership button gets target="_blank". Footer website + IG handle went from plain text to clickable links (IG → instagram.com/<handle>).

● Follow on Instagram

New fuchsia "Follow @handle →" button in the intro area — seen first. Pulls from the org profile; no IG button existed before.

Root cause

The preview is an <iframe srcDoc>, and a srcdoc document inherits its base URL from the parent page. So href="#events" resolved to localhost:3000/#events — clicking it navigated the iframe to the live app, which booted and rendered the Dashboard right in the preview.
