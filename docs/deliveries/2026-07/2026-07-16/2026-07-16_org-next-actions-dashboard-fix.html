🐛 The bug

Org next actions never appeared on the dashboard.

Proved against live data before touching code: 11 orgs have a real next action, and all 16 have next_actions array length 0. The array was always empty, so the dashboard's org loop always produced nothing.

The render path already supported org actions (🏢 badge, click-through to Orgs). That is what proved this was a broken collection step, not an unbuilt feature.

🔍 Root cause

Orgs store one current action in the flat fields next_action / next_action_date, archiving completed ones to next_actions_log. That is the deliberate design, and every other surface follows it.

The dashboard's org loop read only o.next_actions, with none of the flat-field fallback its contact loop has one line earlier.

A second bug hid the first: OrgSchema has no next_actions field, so the two places OrgDetail did write an array were silently stripped by Zod on every save. The schemas-sync trap.

🔴 The trap — why the obvious fix was the wrong one

The tempting fix is adding next_actions to OrgSchema. It would have been actively harmful.

Those writes use mergedNas, which appends but never completes, and no org code path ever marks an array entry completed (completeAction clears only the flat fields). Unblocking the schema would make every touchpoint-with-an-action append an entry that could never be cleared, piling onto the dashboard permanently.

The lesson: when two surfaces disagree, fix the one that departs from the design the other surfaces already share, not the one that merely looks a field short.

✅ The fix — make the dashboard read what orgs actually write

The org loop now reads the flat fields, matching the MCP's predicate exactly, with a comment explaining why there is no array. Removed the two vestigial next_actions writes (already inert) so a future schema change cannot silently reintroduce the accumulation bug.

Before — always empty
(orgs||[]).forEach(o=>{
  (o.next_actions||[])
    .filter(a=>!a.completed&&a.date)
    .forEach(a=>{ ...push... });
});
After — matches how orgs save
(orgs||[]).forEach(o=>{
  if(o.next_action && o.next_action_date){
    allActions.push({type:"org", ...});
  }
});
