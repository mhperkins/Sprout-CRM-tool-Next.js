What shipped

A QR on every slide

Nobody can click a TV. Each slide carries a scannable code to the thing it advertises. Every code was decoded and confirmed to match the URL its caption promises.

Self-contained files

Logo and QRs inlined as base64. The only external dependency is the Lato webfont, which falls back to a system sans offline. Designed 16:9 in vw units, so it scales to any TV.

Built as a loop, then split

Shipped first as one looping deck (12s a slide, cross-fade, infinite wrap), then split into six at Max's request so each can be iterated on alone. Recombine spec is in the README.

One source of truth

The combined deck was deleted, not kept beside the six, to avoid the mirrored-copy drift this project keeps hitting. The six files are authoritative.

Filed under Design

Not Comms, not a new employee. Design already owns every visual surface and self-contained brand HTML; its output standard literally names "slide 16:9". Comms owns the words.

The public/ trap

⚠ Never rename public/

public/ is a reserved Next.js folder and its name is not configurable (unlike Vite's publicDir). Renaming it to sprout-marketing to group the marketing files silently stopped every static asset from being served. Proved against the running dev server rather than asserted:

404 /sprout-sign-in.html 404 /sprout-tv.html 404 /sprout-logo.png

That breaks the live kiosk sign-in page that the printed event QR flyers point at, which would have surfaced at the next event rather than at the keyboard. Reverted; public/ is verified back to its original 7 files. To group marketing files under a URL, use a subfolder (public/marketing/).

Four bugs, all caught by looking

Logo blew out to 1690px wide

Rendered at a 9.77 aspect against its true 2.12. The slide is a flex-direction:column container, so children stretch on the cross axis, which is horizontal.

Fix: align-self:flex-start

The acid rule collapsed to height 0

A childless flex item shrinks to nothing, so the bar under each headline was silently invisible while still measuring 211px wide.

Fix: flex-shrink:0

The split regex ate its own comment

It matched a literal slide tag inside the deck's own head comment, so slide 1 shipped with comment prose in its body and rendered blank. It hit exactly 6 matches by coincidence, so the count assertion passed.

Fix: strip comments first. A passing count is not a passing parse.

Preview tiles didn't fill their frames

A fixed 0.28 scale inside a fluid tile left the frame's background bleeding through the right and bottom edges. Visible only on the light slides, because the dark ones matched the background.

Fix: derive tile width from the scale factor

How to preview

The files live on the Windows box and the browser is on the Mac, over a plain terminal SSH session with no VS Code PORTS panel. Opening a file path does not work. Serve the folder instead:

cd virtual-agency/employees/Design/deliverables/tv-slides
python -m http.server 4000 --bind 0.0.0.0

Then open the contact sheet on the Mac. All six slides live in iframes at a true 1920×1080, scaled down. Click a tile to open it full size, F11 for fullscreen.

http://192.168.50.65:4000/_preview.html ↗

Windows Firewall already has an inbound Allow rule for Python on the Public profile, which is the active network, so no firewall change and no admin is needed. Fallback from anywhere: ssh -N -L 4000:localhost:4000 maxwe@192.168.50.65.
