The blocker

Windows sshd env cache

Mac → Remote-SSH → Windows host. After setx + restarts, the new Trello vars still read empty in-process → Trello 401.

The registry had them; older MCP tokens resolved fine. Cause: Windows OpenSSH freezes the user env at service start and serves that snapshot to every new session.

The fix — sidestep sshd entirely

A registry-read launcher: mcp/trello-launch.cmd

Reads the credentials straight from HKCU\Environment at spawn, then runs the server. No ${VAR} expansion (that pulls the stale env), no secret in the file (safe to commit), no restart or admin needed.
