App feature

📋 Import checklist template

  • New box atop the event Checklist tab: one ＋ Import button per template.
  • Set the event date, click Import → all 23 items land auto-dated off that date.
  • Re-import skips items already present; button disabled until a date is set.
  • Dated items populate the event calendar automatically.
Built from real data

The Sprout N Tell template

  • Lead times derived from the actual Show n Tell (5/19), Sprout n Tell (6/26), and Vol. 3 (7/24) checklists.
  • 23 items, phase-grouped from lineup (−18d) to GAMEDAY (0) to thank-you follow-up (+7).
  • Excluded per Max: ashtray, garbage cans, send-out-zoom-link.
Under the hood — components/CRMManager.jsx
EVENT_CHECKLIST_TEMPLATES {text, offset} → shiftDate(event_date, offset) → importChecklistTemplate() → data.checklist (dated) → calendar renders by date
  • offset = days relative to the event date (negative = before, 0 = event day, +7 = follow-up); shiftDate is noon-anchored to dodge timezone drift. npm run build passes.
Virtual agency — 4th employee

🗓 Event Manager

  • Owns event production: dated prep checklists, run of show, lineup tracking, and +7 close-out.
  • At close-out it hands the recap → Communications, attendee touchpoints → CRM Manager, and new attendees → Outreach.
  • The checklist template lives in its deliverables as the source of truth; the app constant mirrors it (cross-referenced both ways).
